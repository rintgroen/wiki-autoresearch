# Contributing

Contributions are welcome if they keep the skill focused on agent-maintained Obsidian LLM wikis.

Good contribution areas:

- Better candidate scoring heuristics.
- More portable config discovery.
- Additional eval prompts.
- Clearer safety checks for non-git vaults.
- Improved source-claim and benchmark-question formats.

Please keep changes reviewable. This skill is meant to encourage small, auditable wiki improvements, and the repository should follow the same spirit.

Before opening a pull request, run:

```bash
npm run check
```
